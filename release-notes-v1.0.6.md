# Mnemox v1.0.6

发布时间：2026-06-03

## 本次重点更新

### 1. 桌面端稳定性

- 桌面端启用单实例锁，重复打开 Mnemox 时会唤起已有窗口，不再启动第二个应用实例。
- 右侧边栏布局改为写入桌面端稳定偏好文件，重新打开后保留折叠状态、宽度、卡片显示和排序。
- 恢复并同步桌面端打包配置，桌面版本更新到 `1.0.6`。

### 2. 聊天与联网搜索

- 聊天区隐藏手动联网搜索开关，默认按当前后端搜索能力启用网页搜索。
- 官方 OpenAI provider 保留 Responses API 内置搜索路径。
- 非官方或不支持内置搜索的供应商会降级到应用层网页搜索，再把检索结果作为上下文提供给模型。
- 供应商设置页新增“搜索模型”和“验证模型”流程，支持自动查询当前供应商可用模型并通过下拉框选择后验证。
- 聊天报错提示补充了 AI 提供商设置指引，并明确联网搜索默认复用当前供应商 API Key，不需要单独配置搜索 Key。

### 3. 工作台体验

- 新增独立番茄工作法页面，当前计时状态通过全局组件持续展示。
- 计划页改为文档工作台，今日任务可以直接从右侧栏跳转到当天计划。
- 笔记页调整为三栏工作台，并修复空文件夹切换时残留上一条笔记的问题。
- 设置、账号入口、会话侧栏和 AI 设置抽屉的视觉与交互进一步统一。

### 4. 安全与多用户隔离

- 补强 AI 设置、聊天、系统状态等接口的多用户隔离校验。
- 优化 AI provider 错误提示，减少原始异常泄露给前端。
- 新增系统 onboarding 状态接口，减少仅依赖前端本地标记的首次引导行为。

## 回归验证

已完成以下验证：

- `backend`
  - `venv\Scripts\python.exe -m pytest`
- `frontend`
  - `npm run test`
  - `npm run build`
- `desktop`
  - `node --test src\*.test.js`

## 对用户的直接影响

- 双击或重复启动桌面端时，只会保留一个 Mnemox 实例。
- 右侧边栏的自定义布局在重新打开桌面端后不会被随机端口导致的 `localStorage` 变化重置。
- 聊天默认具备网页检索能力，用户不需要额外打开搜索按钮。
- 供应商模型不需要再手动硬编码输入，搜索后即可选择并验证。
- 番茄、计划和笔记工作流更接近长期使用的学习工作台。
