# UI 和功能优化方案

## 当前问题分析

### UI 布局问题

#### 1. 信息过载 - 主界面按钮太多
**当前状态：**
```
顶部导航栏有 11 个按钮：
- 错题本页
- 复习中心页
- 目标任务页
- 笔记页
- 记忆管理页
- 今日驾驶舱
- 掌握度地图
- 进度引擎
- 番茄统计页
- AI 设置
- 折叠侧边栏
```

**问题：**
- 用户一眼看不过来，不知道该点哪个
- 很多功能可能用户根本不需要
- 破坏了"AI 对话"这个核心功能的聚焦感

#### 2. 左侧边栏功能混杂
**当前状态：**
- Tab 1: 对话列表（包含项目管理、对话管理）
- Tab 2: 资料列表（包含上传、搜索、筛选、项目绑定）
- Tab 3: 番茄钟（计时器、统计）
- Tab 4: 日历计划（周计划、日计划）

**问题：**
- 4 个 Tab 切换频繁，用户容易迷失
- 番茄钟和日历计划与"对话"功能关联不强
- 资料管理功能过于复杂（项目筛选、批量绑定等）

#### 3. 右侧边栏信息密度过高
**当前状态：**
- 番茄钟控制面板
- 今日统计
- 错题预览
- 复习任务
- 学习仪表盘
- 记忆指标

**问题：**
- 太多卡片堆叠，用户不知道该看哪个
- 很多信息可能不是用户当前需要的
- 分散了对话的注意力

#### 4. 项目功能定位不清
**当前状态：**
- 项目可以关联资料
- 项目有默认指令
- 对话可以属于项目
- 资料可以按项目筛选

**问题：**
- 用户不理解"项目"是什么
- 项目-资料-对话的关系复杂
- 很多操作需要在多个地方切换

---

## 优化方案

### 方案概述

**核心理念：**
- 聚焦核心功能：AI 对话 + 资料学习
- 简化信息层级：减少 Tab、减少按钮、减少选项
- 渐进式展示：常用功能优先，高级功能隐藏

---

### 第一阶段：UI 简化（不改功能逻辑）

#### 1.1 顶部导航栏优化

**改动：**
```
之前：11 个按钮平铺
之后：3 个主要按钮 + 1 个下拉菜单

[折叠侧边栏] [AI 学习助手] [设置] [更多 ▼]

更多菜单包含：
├─ 学习工具
│  ├─ 番茄统计
│  ├─ 错题本
│  └─ 复习中心 (badge)
├─ 知识管理
│  ├─ 笔记
│  ├─ 记忆管理
│  └─ 掌握度地图
└─ 数据分析
   ├─ 今日驾驶舱
   ├─ 进度引擎
   └─ 目标任务
```

**好处：**
- 视觉清爽，用户不会被按钮淹没
- 保留所有功能，只是收起来
- 用户可以专注于对话

#### 1.2 左侧边栏简化

**改动：**
```
之前：4 个 Tab（对话、资料、番茄钟、日历）
之后：2 个 Tab（对话、资料）

Tab 1: 对话
├─ 项目分组（可折叠）
│  ├─ 项目 A
│  │  ├─ 对话 1
│  │  └─ 对话 2
│  └─ 项目 B
└─ 未分类对话

Tab 2: 资料
├─ 搜索框
├─ 上传按钮
├─ 筛选器（简化版）
│  ├─ 全部资料
│  └─ 当前项目资料（如果有项目）
└─ 资料列表
```

**移除的功能去哪了：**
- 番茄钟 → 移到右侧边栏顶部（可折叠卡片）
- 日历计划 → 移到独立页面（通过"更多"菜单访问）

**好处：**
- 只保留与对话强相关的功能
- 减少 Tab 切换次数
- 资料管理更直观

#### 1.3 右侧边栏优化

**改动：**
```
之前：6 个卡片平铺
之后：可折叠的手风琴面板

[番茄钟] ▼ （默认展开）
├─ 计时器
└─ 今日统计

[学习提醒] ▼ （默认折叠）
├─ 待复习任务 (3)
└─ 错题待巩固 (5)

[对话资料] ▼ （默认折叠）
└─ 当前对话使用的资料列表

[记忆指标] ▼ （默认折叠）
└─ AI 记住的关键信息
```

**好处：**
- 用户可以自己控制显示什么
- 减少视觉噪音
- 保留所有功能

#### 1.4 中间对话区优化

**改动：**
```
之前：顶部有很多 Tag 显示状态
之后：简化为 1 行状态栏

[对话 #123] [项目: Python 学习] [教练模式] [正在生成...]

资料选择器：
之前：在侧边栏勾选，不直观
之后：在输入框上方显示"已选资料"标签，点击可管理
```

**好处：**
- 对话区更宽敞
- 资料选择更直观
- 状态信息更清晰

### 第二阶段：功能逻辑优化

#### 2.1 重新定义"项目"

**新定义：**
- 项目 = 学习主题（如"考研数学"、"Python 入门"）
- 每个项目有自己的资料库
- 每个项目有自己的对话历史

**改动：**
```sql
-- 保持现有表结构，但改变使用方式

ChatProject:
- name: 项目名称（必填）
- description: 项目描述（可选）
- default_instructions: 项目学习指令（可选）
- color: 项目颜色标识

ChatProjectMaterial:
- 资料上传时必须选择项目
- 资料可以属于多个项目（通过复制关联）

ChatConversation:
- 对话创建时必须选择项目
- 对话移动到其他项目时，保留原有资料和指令
```

**用户体验：**
1. 用户创建项目："考研数学"
2. 上传资料时选择项目："高等数学.pdf" → 考研数学
3. 创建对话时选择项目：新对话 → 考研数学
4. AI 自动使用该项目的资料回答问题

#### 2.2 简化资料管理

**改动：**
```
之前：
- 资料可以不属于任何项目
- 需要手动绑定到项目
- 有"未分类"项目的概念

之后：
- 资料上传时必须选择项目
- 如果没有项目，引导用户先创建项目
- 移除"未分类"项目
- 移除"批量绑定"功能（因为上传时已选择）
```

**好处：**
- 用户不会有"孤儿资料"
- 资料管理更清晰
- 减少操作步骤

#### 2.3 优化对话-资料关联

**改动：**
```
之前：
- 对话自动使用项目的所有资料
- 用户不知道 AI 在用哪些资料
- 无法临时添加/移除资料

之后：
- 对话默认使用项目的所有资料
- 在输入框上方显示"当前使用的资料"标签
- 用户可以点击标签，临时添加/移除资料
- 临时修改不影响项目设置
```

**UI 示例：**
```
┌─────────────────────────────────────────┐
│ 当前使用资料：                           │
│ [高等数学.pdf ×] [线性代数.pdf ×] [+ 添加] │
└─────────────────────────────────────────┘
┌─────────────────────────────────────────┐
│ 输入消息...                              │
└─────────────────────────────────────────┘
```

#### 2.4 移除冗余功能

**建议移除：**
1. `archive_unassigned_materials` 端点（因为资料必须属于项目）
2. 资料的"项目筛选"开关（因为已经在项目视图中）
3. 对话的"项目指令动态跟随"（改为创建时继承）

**建议保留但简化：**
1. 批量资料关联端点（用于项目设置页面）
2. 资料的 content_status（用于显示提取失败警告）

### 第三阶段：新增便捷功能

#### 3.1 快速切换项目

**新增：**
```
顶部导航栏添加项目切换器：

[AI 学习助手] [项目: 考研数学 ▼]

下拉菜单：
├─ 考研数学 ✓
├─ Python 入门
├─ 英语四级
├─ ──────────
└─ 管理项目...
```

**好处：**
- 快速切换学习主题
- 不需要去侧边栏找项目
- 更符合用户心智模型

#### 3.2 智能资料推荐

**新增：**
```
当用户提问时，AI 分析问题并推荐相关资料：

用户："线性代数的特征值怎么求？"

AI 检测到：
- 当前项目有《线性代数.pdf》
- 但用户没有选择这个资料

显示提示：
┌─────────────────────────────────────────┐
│ 💡 检测到相关资料：线性代数.pdf          │
│ [使用这个资料回答] [忽略]                │
└─────────────────────────────────────────┘
```

#### 3.3 对话模板

**新增：**
```
创建对话时可以选择模板：

├─ 空白对话
├─ 知识学习（AI 会详细解释概念）
├─ 习题辅导（AI 会引导思考，不直接给答案）
├─ 快速答疑（AI 简洁回答）
└─ 自定义...

模板会设置不同的 system prompt
```

## 需要确认的问题

1. **项目必选 vs 可选**
   - 方案建议：资料和对话都必须属于项目
   - 你的想法：是否同意？还是希望保留"无项目"的选项？

2. **番茄钟的位置**
   - 方案建议：移到右侧边栏或独立页面
   - 你的想法：番茄钟对你来说重要吗？需要一直可见吗？

3. **日历计划的位置**
   - 方案建议：移到独立页面
   - 你的想法：日历计划使用频率高吗？

4. **顶部导航栏的功能**
   - 方案建议：只保留 3-4 个主要按钮，其他收到"更多"菜单
   - 你的想法：哪些功能是你最常用的？

5. **资料上传流程**
   - 方案建议：上传时必须选择项目
   - 你的想法：是否接受？还是希望可以"先上传，后分类"？
