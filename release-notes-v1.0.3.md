# Mnemox v1.0.3

发布时间：2026-05-23

## 本次重点更新

### 1. 桌面版登录记忆与自动登录

- 新增桌面安全凭据存储：Electron 通过系统加密存储保存用户名和密码，不再把密码放进浏览器本地存储。
- 登录页新增：
  - `记住密码`
  - `下次自动登录`
  - `清除已保存登录信息`
- 普通退出登录只会清掉当前 token，不会误删已保存的桌面凭据；如果需要彻底清除，可在登录页主动清除。

### 2. 托盘常驻与桌面提醒

- 桌面版关闭主窗口后不再直接退出，而是最小化到系统托盘后台常驻。
- 番茄钟开始、暂停、恢复、完成会同步到 Electron 主进程。
- 即使主窗口关闭，番茄钟到时仍会：
  - 弹出系统通知
  - 播放提示音
  - 点击通知可重新打开主窗口

### 3. API 错误提示与可诊断性增强

- 修复多处前端服务层把真实后端异常吞成 `null` / `[]` 的问题。
- 现在以下高频模块在失败时会明确提示，而不是显示成“空列表”或“没反应”：
  - 今日概览 / 学习仪表盘
  - AI 设置
  - Pomodoro
  - 对话与项目
  - 目标与任务
  - 复习中心
  - Agent 写入草稿 / 执行
  - 激励语录与相关设置
- 后端不可用时，会统一显示可读错误：`后端服务不可用，请确认 Mnemox 后端已启动`

### 4. 更新发布链路补齐

- 桌面打包运行时现在内置公开更新清单地址：
  - `https://raw.githubusercontent.com/wlohf/Mnemox/main/release-manifest/latest.json`
- 应用内“检查更新”除了 Electron 的 GitHub Release 检测外，也会显示统一的版本说明、发布时间和下载地址。
- 新增发布脚本：
  - `scripts/publish_desktop_release.ps1`
- 以后每次发布只要更新版本、重打包、发 GitHub Release，并同步更新 `release-manifest/latest.json`，本地安装包就能直接查询到并更新。

## 对用户的直接影响

- 本地安装的桌面版现在可以稳定检测 GitHub Release 更新。
- 登录体验更顺手，桌面版支持真正的“记住密码 + 自动登录”。
- 关闭窗口后番茄钟不会丢提醒。
- 出问题时更容易看到真实原因，不会再经常误以为“只是没数据”。
