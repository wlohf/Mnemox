## ⚡ 快速启动（推荐）

### 方法一：使用批处理脚本（.bat）

**最简单！双击即可运行**

1. 找到项目根目录的 `start.bat` 文件
2. **双击运行**
3. 等待几秒，两个窗口会自动打开：
   - **后端窗口**：显示 API 服务日志
   - **前端窗口**：显示前端开发服务器
4. 浏览器会自动打开 `http://localhost:5173`

```
E:\xyleisure\StudyAssitant\
├── start.bat          👈 双击这个文件
├── start.ps1
├── stop.bat
└── ...
```

---

### 方法二：使用 PowerShell 脚本（.ps1）

**功能更强大，彩色输出**

1. 找到项目根目录的 `start.ps1` 文件
2. **右键点击** → 选择 **"使用 PowerShell 运行"**
3. 如果提示权限错误，请先执行：
   ```powershell
   Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
   ```
4. 等待启动完成

---

## 🔧 手动启动

如果启动脚本无法使用，可以手动启动：

### 第一步：启动后端

打开终端（PowerShell 或 CMD），执行：

```bash
# 进入后端目录
cd E:\xyleisure\StudyAssitant\backend

# 启动后端服务
python -m app.main
```

**看到以下输出说明后端启动成功**：
```
INFO:     Uvicorn running on http://127.0.0.1:8000
INFO:     Application startup complete.
```

---

### 第二步：启动前端

**另开一个新终端**，执行：

```bash
# 进入前端目录
cd E:\xyleisure\StudyAssitant\frontend

# 启动前端服务
npm run dev
```

**看到以下输出说明前端启动成功**：
```
VITE v5.x.x  ready in xxx ms

➜  Local:   http://localhost:5173/
➜  Network: use --host to expose
```

---

### 第三步：打开浏览器

访问：**http://localhost:5173**

## 🛑 停止服务

### 方法一：使用停止脚本（推荐）

1. 双击运行 `stop.bat`

1. 按 `Ctrl + Shift + Esc` 打开任务管理器
2. 找到 `python.exe` 和 `node.exe` 进程

## ❓ 常见问题

### Q1: 双击 `start.bat` 后窗口一闪而过？

**原因**：可能是 Python 或 Node.js 未安装或未添加到环境变量

**解决方法**：
1. 右键点击 `start.bat` → **"编辑"**
2. 在第一行下面添加 `pause`，查看错误信息
3. 或者手动启动查看详细错误

---

### Q2: 提示端口已被占用？

**错误信息**：
```
Address already in use
EADDRINUSE: address already in use :::8000
```

**解决方法**：

#### 方法一：使用停止脚本
```bash
双击 stop.bat
```

#### 方法二：手动查找并关闭占用端口的进程

**后端端口 8000**：
```powershell
# 查找占用 8000 端口的进程
netstat -ano | findstr :8000

# 结束进程（替换 PID 为实际进程号）
taskkill /F /PID <PID>
```

**前端端口 5173**：
```powershell
# 查找占用 5173 端口的进程
netstat -ano | findstr :5173

# 结束进程
taskkill /F /PID <PID>
```

---

### Q3: 后端启动报错 `ModuleNotFoundError`？

**错误信息**：
```
ModuleNotFoundError: No module named 'fastapi'
```

**解决方法**：安装后端依赖

```bash
cd backend
pip install -r requirements.txt
```

---

### Q4: 前端启动报错 `command not found: npm`？

**原因**：Node.js 未安装或未添加到环境变量

**解决方法**：
1. 安装 Node.js（已安装可跳过）
2. 安装前端依赖：
   ```bash
   cd frontend
   npm install
   ```

---

### Q5: 前端页面空白或报错？

**可能原因**：
- 后端未启动
- 前端依赖未安装
- 浏览器缓存问题

**解决方法**：
1. 确认后端和前端都已启动
2. 按 `Ctrl + Shift + R` 强制刷新浏览器
3. 检查浏览器控制台（F12）查看错误信息

---

### Q6: PowerShell 脚本无法运行？

**错误信息**：
```
无法加载文件，因为在此系统上禁止运行脚本
```

**解决方法**：修改执行策略

```powershell
# 以管理员身份打开 PowerShell，执行：
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

# 输入 Y 确认
```

---

### Q7: 浏览器没有自动打开？

**手动打开浏览器**，访问：http://localhost:5173

---

### Q8: 如何查看后端日志？

**方法一**：查看启动脚本打开的"学习助手-后端"窗口

**方法二**：手动启动后端并查看输出
```bash
cd backend
python -m app.main
```

---

### Q9: 如何修改后端端口？

1. 打开 `backend/.env` 文件
2. 修改 `PORT=8000` 为其他端口，如 `PORT=8080`
3. 重启后端服务

---

### Q10: 如何修改前端端口？

1. 打开 `frontend/vite.config.ts`
2. 添加或修改 `server` 配置：
   ```typescript
   export default defineConfig({
     server: {
       port: 3000, // 改为你想要的端口
     },
     // ... 其他配置
   })
   ```
3. 重启前端服务

## 💡 高级技巧

### 1. 开机自动启动

如果你想让学习助手开机自动启动：

1. 按 `Win + R`，输入 `shell:startup`
2. 将 `start.bat` 的快捷方式复制到这个文件夹

### 2. 创建桌面快捷方式

1. 右键点击 `start.bat`
2. 选择 **"发送到"** → **"桌面快捷方式"**
3. 可以右键快捷方式 → **"属性"**，修改图标

### 3. 使用 VSCode 集成终端

如果使用 VSCode 开发：
1. 打开项目
2. 按 `Ctrl + ` ` 打开终端
3. 点击终端右上角 **"+"** 旁的下拉箭头
4. 选择 **"拆分终端"**
5. 一个运行后端，一个运行前端
